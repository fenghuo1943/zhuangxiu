# 装修网站开发指导文档（优化版）

## 0. AI编程基础规则（前置必读）

开发此项目时应严格遵循以下规则：

### 0.1 首要原则
- **工具优先**：页面是项目管理工具，非营销网站。保证清晰、稳定、高频操作效率。
- **先首页后扩展**：严格按优先级开发（首页 → 流程页 → 比价页 → 记账页），不要提前开发未提供参考图的复杂页面。
- **响应式并行**：桌面端（≥1024px）和移动端（<768px）同时设计和实现，不是先做桌面再改手机。
- **禁止虚假实现**：不要把未实现的功能伪装成已实现；可以先做静态入口或禁用态。

### 0.2 组件复用与代码质量
- **复用第一**：优先复用已有组件（Card、Button、IconButton、Badge 等），不要在每个模块重复写样式。
- **模型一致**：数据模型应遵循全局 schema（见第 2 章），不要在各模块创建私有模型。
- **无数据默认值**：所有数字显示都要有默认值（`0.00` 或 `--`），**禁止出现 `NaN`、`undefined`**。
- **列表必有空状态**：每个列表、分组、面板都要有自然的空状态文案和建议操作。

### 0.3 开发流程与验证
1. 阅读本文档，理解项目定位和当前模块需求。
2. 查看现有代码结构，判断需要新增的组件和数据模型。
3. 实现静态页面（布局 + 样式），同时检查 375px 和 1440px 两个断点。
4. 添加基础交互（状态管理、事件处理），运行项目并验证页面。
5. 实现本地存储持久化（`localStorage`），确保刷新后能恢复数据。
6. 按照验收清单逐项检查，对标参考图进行微调。

### 0.4 禁止事项
- 不要做大面积渐变英雄区或营销型大横幅。
- 不要在工具界面中堆叠大量解释性文字。
- 不要使用过度装饰、厚重阴影、高饱和色彩。
- 不要在一个页面同时引入无关的第三方样式库（与已有风格不兼容）。
- 不要实现云同步或后端 API 集成（首版使用本地存储）。

### 0.5 性能与可访问性
- 表单输入应限制输入类型，自动千分位格式化，失焦时校验。
- 所有可点击元素保留 hover、active、disabled 状态。
- 使用语义化 HTML（button、header、nav、section、ul/li）。
- 加入 `aria-expanded`、`aria-controls`、`role` 等属性支持屏幕阅读器。
- 操作按钮触控目标不小于 44×44px（移动端）。

## 1. 全局设计系统

### 1.1 颜色系统
- **主色**：珊瑚红 `#E45B3F` / 橙红 `#E88670`（主按钮、重要入口、强调文字）
- **辅助色**：
  - 薄荷绿 `#5FB27C`（成功、确认、预算充足）
  - 浅蓝 `#4A90E2`（信息、流程进度）
  - 浅橙 `#FFD166`（警告、预算预警）
  - 浅红 `#EF476F`（错误、超支警告）
- **中立色**：
  - 白色 `#FFFFFF`（卡片背景）
  - 浅暖灰 `#F5F3F0` 或米白 `#FFFBF7`（页面背景）
  - 深灰 `#666666`（正文文字）
  - 浅灰 `#CCCCCC`（辅助文字、边框）

### 1.2 排版
- **中文字体**：优先使用系统字体堆栈
  ```css
  font-family: -apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, "Helvetica Neue", Arial, sans-serif;
  ```
- **字号与行高**：
  - 标题 H1：24px / 1.5
  - 标题 H2：20px / 1.4
  - 标题 H3：16px / 1.4
  - 正文：14px / 1.6
  - 小文本：12px / 1.5

### 1.3 间距
- 基础单位：8px
- 卡片内边距：16px（移动端）/ 20px（桌面端）
- 卡片间距：12px（移动端）/ 16px（桌面端）
- 页面顶部留白：16px（移动端）/ 24px（桌面端）

### 1.4 圆角与阴影
- 卡片圆角：12px（移动端）/ 16px（桌面端）
- 按钮圆角：8px
- 轻阴影：`0 2px 8px rgba(0, 0, 0, 0.08)`
- 中阴影：`0 4px 16px rgba(0, 0, 0, 0.12)`
- 禁止使用厚重阴影（> 0.2 alpha）

### 1.5 响应式断点
- **移动端**：< 768px（单列流式布局）
- **平板端**：768px ~ 1023px（过渡布局）
- **桌面端**：≥ 1024px（多列布局）
- **最大内容宽度**：1360px ~ 1440px（桌面端）

## 2. 全局数据模型架构

### 2.7 状态管理建议
- **局部状态**：UI 展开收起状态、表单输入值等，用 React `useState` 或 Vue `ref`。
- **全局状态**：Project、Budget、Stage、Todos、Expenses 等，用 Context + useReducer 或 Pinia/Vuex。
- **持久化**：写入 `localStorage` 时以 `${projectId}_${modelName}` 为 key。

## 3. 项目定位与目标

### 3.1 项目性质
本项目不是传统装修公司展示官网，而是面向个人或家庭装修过程的**项目管理工具**。首页应以"装修项目仪表盘"为核心，帮助用户快速查看预算、阶段、待办、采购、记账和流程提醒。

**开发时优先保证工具型产品的清晰、稳定和高频操作效率，不要做成营销落地页。**

### 3.2 核心功能
1. **预算管理**：设置总预算、按分类分配、实时跟踪支出。
2. **阶段进度**：展示装修流程、标记完成、自动计算进度。
3. **待办清单**：按阶段分组待办，支持新增、标记完成、删除。
4. **记账系统**：按分类记录支出，支持多渠道导入导出。
5. **采购管理**：采购参考库、采购比价、待购清单。
6. **流程指南**：分阶段展示施工标准、验收要点、攻略文章。

### 3.3 用户场景
- **计划阶段**：设置总预算、了解装修流程、制定待办计划。
- **进行阶段**：记录支出、检查预算余额、标记流程进度、采购比价。
- **完成阶段**：查看支出统计、导出记账数据、总结项目。

## 5. 模块一：首页（Dashboard）

### 5.1 首页目标
1. 让用户知道当前装修项目处于什么阶段。
2. 让用户知道今天需要处理什么。
3. 让用户快速进入预算、待办、采购、记账和流程模块。
4. 让用户看到预算设置状态、剩余金额、支出和使用率。
5. 在桌面端展示完整信息，在移动端保留最高频、最关键的信息。

### 5.2 首页模块拆分

#### 5.2.1 顶部导航 Header（复用组件）
**桌面端元素**：
- 品牌/Logo：小装家
- 副标题：清晰装修管家
- 导航链接：首页（激活态）、采购、比价、记账、流程、工具
- 公告按钮
- 当前项目选择器
- 新建项目按钮
- 账号入口

**移动端元素**：
- 首页按钮 / Logo
- 账号入口
- 切换/菜单按钮
- 公告按钮

**组件名**: `DesktopHeader`、`MobileHeader`（或统一为 `Header` 支持 props 区分）

---

#### 5.2.2 今日装修提醒 ReminderCard

**内容结构**：
```typescript
{
  title: string;        // "今日装修提醒"
  description: string;  // 根据预算和阶段状态生成
  actions: Action[];    // [{ label: "新增待办", onClick }, ...]
```

**按钮**：

#### 5.2.6 预算设置与阶段分配 BudgetPanel（核心交互）

**显示内容**：
- 统计行：已支出 / 总预算 / 剩余 / 使用率
- 总预算输入框（数字输入，自动千分位）
- 各阶段预算分配（可视化条形图 + 输入框）
- 各阶段支出与占比

**交互规则**：
- 用户输入总预算后实时更新剩余和使用率
- 各阶段预算总和不能超过总预算（超过时提示）
- 预算为 0 时显示"未设置"或占位符
- 没有数据时所有数字显示 `0.00` 或 `--`

**预算分类**（5 类）：
1. 硬装工程
2. 主材选购
3. 设备系统
4. 软装家电
5. 服务杂项
