## 1. 系统概述

**装修手记**是一个面向个人装修管理的一站式工具，帮助你清晰掌控装修全过程。系统采用前后端分离架构，核心数据存储在浏览器本地（localStorage），登录后支持云端同步。

### 核心功能一览

| 模块 | 说明 |
|------|------|
| 📊 **首页仪表盘** | 今日提醒、预算概览、待办事项、阶段进度、快捷入口 |
| 🗺️ **装修流程** | 22 步新房 / 9 步旧房流程管理，含施工标准 & 避坑指南 |
| 🛒 **采购清单** | 27 项预置参考材料清单，支持自定义添加 |
| 💰 **比价管理** | 按品类—型号—渠道三级管理报价，一键同步到采购清单 |
| 📝 **记账管理** | 5 大分类记账，支持 CSV/JSON 导入导出，BOM 物料单 |
| 🔧 **工具集** | 装修预算计算器、工期估算器 |
| 💾 **数据管理** | 本地导出/导入 JSON，云端上传/恢复 |
| 🔐 **多端同步** | 注册登录后支持多设备数据同步 |

## 3. 环境要求与部署

### 系统要求

- **Python** ≥ 3.10
- **Node.js** ≥ 18
- **MySQL** ≥ 8.0（可使用已有的远程实例）
- **包管理器**：pip + npm

### 3.1 后端部署

#### Step 1：进入后端目录

```bash
cd backend
```

#### Step 2：创建虚拟环境（可选但推荐）

```bash
python -m venv venv
source venv/bin/activate       # Linux/Mac
venv\Scripts\activate          # Windows
```

#### Step 3：安装依赖

```bash
pip install -r requirements.txt
```

#### Step 4：配置数据库

编辑 [backend/config.py](../backend/config.py)（或通过环境变量覆盖）：

| 环境变量 | 默认值 | 说明 |
|----------|--------|------|
| `DATABASE_URL` | `mysql+aiomysql://renovation:密码@192.168.31.146:3307/renovation` | MySQL 连接串 |
| `JWT_SECRET` | 内置 dev secret | JWT 签名密钥（**生产环境务必更换**） |

数据库（`renovation`）需提前在 MySQL 中创建。所有表结构由 SQLAlchemy 在首次启动时自动建表，无需手动迁移。

#### Step 5：启动后端

```bash
uvicorn backend.main:app --host 0.0.0.0 --port 8003 --reload
```

- `--reload`：开发模式下代码变更自动重启
- 启动后访问 `http://localhost:8003/docs` 可查看 Swagger API 文档

### 3.2 前端部署

#### Step 1：进入前端目录

```bash
cd frontend
```

#### Step 2：安装依赖

```bash
npm install
```

#### Step 3：配置 API 地址

API 基础地址配置在 [frontend/src/api/client.ts](../frontend/src/api/client.ts)：

```typescript
const API_BASE = "http://localhost:8003";
```

如需修改后端地址，改动此处即可。

#### Step 4：启动开发服务器

```bash
npm run dev
```

默认运行在 `http://localhost:5173`。Vite 支持热更新（HMR），修改代码即时生效。

#### Step 5：构建生产版本

```bash
npm run build
```

构建产物输出到 `frontend/dist/`，可直接部署到任意静态文件服务器。

### 3.3 生产环境部署

#### 后端

```bash
# 使用生产级 ASGI 服务器
uvicorn backend.main:app --host 0.0.0.0 --port 8003 --workers 4
```

**生产环境注意事项：**

- 修改 `JWT_SECRET` 为安全的随机字符串：
  ```bash
  export JWT_SECRET=$(python -c "import secrets; print(secrets.token_urlsafe(32))")
  ```
- 配置防火墙仅开放必要端口
- 生产环境建议使用 Nginx 反向代理，并配置 HTTPS

#### 前端

将 `frontend/dist/` 部署到 Nginx 或其他静态文件服务。Nginx 基础配置示例：

```nginx
server {
    listen 80;
    server_name your-domain.com;
    root /path/to/frontend/dist;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }
}
```

> `/index.html` 的 fallback 是必需的，因为前端使用 React Router 的客户端路由。

## 4. 功能模块说明

### 4.1 用户系统

系统支持两种使用模式：

**离线模式（默认）：** 无需注册登录即可使用全部本地功能。数据存储在浏览器 localStorage 中，不同设备间数据不互通。

**在线模式：** 注册账号后登录，可使用云端数据同步功能，实现多设备数据互通。

#### 注册

1. 点击页面右上角「登录」→「立即注册」
2. 填写用户名（2–50 字符）、邮箱、密码（至少 6 位）
3. 提交后自动登录并跳转到首页

#### 登录

1. 点击右上角「登录」
2. 输入用户名和密码
3. 登录成功后返回首页

#### 账号管理

登录后点击右上角用户图标可进入账号页面。

### 4.2 首页仪表盘

首页是装修管理的核心入口，包含以下模块：

#### 📌 今日装修提醒

- 根据当前预算和阶段状态生成提醒文案
- 显示待办数量、待购材料数量、预算余额等关键指标
- 三个快捷操作按钮：**新增待办**、**记一笔**、**看流程**

#### 📊 预算面板

- **总预算**：点击数字可编辑总预算金额
- **分类分配**：5 个预算分类（硬装工程 / 主材选购 / 设备系统 / 软装家电 / 服务杂项），每个分类可拖动滑块调整分配金额
- **已支出**：自动根据记账数据汇总，红色进度条显示
- **余额 & 使用率**：实时计算

| 分类 | 建议占比 | 说明 |
|------|----------|------|
| 硬装工程 | 32% | 拆改、水电、泥瓦、木工、油漆等人工+辅材 |
| 主材选购 | 28% | 瓷砖、地板、门窗、涂料等 |
| 设备系统 | 15% | 中央空调、新风、地暖、净水等 |
| 软装家电 | 18% | 家具、灯具、窗帘、家电等 |
| 服务杂项 | 7% | 设计费、监理费、保洁、搬家等 |

#### ✅ 待办事项

- **新增待办**：输入标题，选填所属阶段和截止日期
- **模式切换**：详细模式（含阶段/日期）和简洁模式（仅标题）
- **完成标记**：点击复选框标记完成，已完成项目折叠显示
- **删除**：点击 × 按钮删除

#### 📈 阶段进度路线

横向展示 22 个装修阶段，已完成的显示为绿色，当前阶段高亮，未开始为灰色。点击任一阶段可跳转到流程页面查看详情。

#### 🎯 今日重点（桌面端）

- 优先处理项提示
- 未完成待办 / 待购材料数量
- 预算余额 / 使用率

#### 其他快捷模块

- **采购概要**：已选采购项数量
- **支出概要**：最近 5 笔支出记录
- **当前阶段指南**：当前阶段名称和说明
- **快捷入口**：流程 / 采购 / 工具
- **数据管理**：新建项目 / 导出 / 导入 / 云端同步

### 4.3 装修流程管理

支持新房装修（22 步）和旧房翻新（9 步）两套流程。

#### 流程切换

在页面顶部可切换「新房装修」或「旧房翻新」流程类型。切换后进度独立保存。

#### 阶段操作

每个阶段卡片包含：

- **预估工期**：理论天数范围
- **详细说明**：阶段内容和注意事项
- **施工标准**：规范要求
- **验收标准**：关键检查项
- **攻略文章**：参考指南
- **避坑指南**：常见问题和陷阱

#### 进度追踪

- 点击「标记完成」勾选已完成阶段
- 进度条实时显示完成百分比
- 首页的阶段路线图同步更新

#### 自定义排序

1. 点击左侧「编辑排序」进入排序模式
2. 使用上下箭头调整阶段顺序
3. 点击「完成」或「恢复默认」退出

### 4.4 采购清单

系统预置了 27 项常见装修材料作为参考，分布在 5 大阶段分类中。

#### 预置参考清单

| 阶段父类 | 子分组 | 材料举例 |
|----------|--------|----------|
| 硬装工程 | 水电材料、防水材料、水泥砂浆 | 电线、水管、防水涂料、水泥 |
| 主材选购 | 瓷砖、地板、门窗 | 地砖、木地板、防盗门 |
| 设备系统 | 暖通、净水 | 中央空调、前置过滤器 |
| 软装家电 | 大家电、照明 | 冰箱、吸顶灯 |

### 4.5 比价管理

用于同一材料在不同渠道的报价对比，帮助找到最优采购方案。

#### 数据组织（三级结构）

```
品类（如"瓷砖"）
  ├── 型号（如"东鹏 800×800 灰色通体砖"）
  │     ├── 报价：京东 ¥128
  │     ├── 报价：线下专卖店 ¥135
  │     └── 报价：淘宝 ¥119
  └── 型号（如"诺贝尔 600×1200 柔光砖"）
        ├── 报价：天猫 ¥298
        └── 报价：居然之家 ¥320
```

#### 操作方式

1. **新建品类**：输入名称，选择 emoji 图标
2. **添加型号**：在品类下添加具体型号，填写名称、规格、备注、数量
3. **添加报价**：对每个型号添加渠道报价（渠道名 + 价格 + 链接）
4. **同步到采购**：点击型号旁的云朵图标，将型号信息同步到采购清单
5. **导入/导出**：支持 CSV 格式文件导入导出

### 4.6 记账管理

全面记录装修期间的所有支出，支持多种筛选和导入导出。

#### 支出一条龙

- **记一笔**：填写标题、金额、分类、日期、状态、备注
- **支付状态**：已支付 / 预付 / 未支付 / 已退款
- **阶段标注**：可选关联到具体装修阶段

#### 分类体系

| 分类 ID | 分类名 | 说明 |
|---------|--------|------|
| `hard` | 硬装工程 | 人工费、辅材 |
| `material` | 主材选购 | 瓷砖、地板、门窗等 |
| `equipment` | 设备系统 | 空调、新风、地暖等 |
| `soft` | 软装家电 | 家具、灯具、家电 |
| `service` | 服务杂项 | 设计、保洁、搬家 |

#### 筛选与搜索

- **状态筛选**：全部 / 已支付 / 预付 / 未支付
- **关键词搜索**：按标题和备注搜索
- **阶段筛选**：展开分类后可按阶段过滤

#### 导入导出

- **导出 CSV**：生成 Excel 兼容的 BOM 物料单
- **导出 JSON**：全量数据备份
- **导入 CSV**：批量导入支出记录，预览模式显示校验结果和错误行
- **导入 JSON**：恢复备份数据

#### 撤销删除

删除支出后底部出现撤销提示，5 秒内可点击恢复。

### 4.8 数据备份与同步

#### 本地数据管理（无需登录）

| 操作 | 说明 |
|------|------|

| 操作 | 说明 |

> ⚠️ 注意：云同步采用整项目覆盖策略，不会智能合并。上传前请确认本地数据是最新的。

## 5. 后端 API 接口文档

### 公共说明

- **基础地址**：`http://localhost:8003`
- **认证方式**：HTTP Bearer Token（JWT，有效期 7 天）
- **需认证的接口**：除注册/登录/健康检查/采购参考清单外，均需在请求头中携带 `Authorization: Bearer <token>`
- **Content-Type**：`application/json`（文件上传使用 `multipart/form-data`）
- **交互式文档**：启动后端后访问 `http://localhost:8003/docs`（Swagger UI）

---

### 5.1 健康检查

#### `GET /api/health`

无需认证。

**响应示例：**

```json
{
  "status": "ok",
  "service": "装修手记 API"
}
```

---

### 5.2 认证接口

**路由前缀：** `/api/auth`

#### `POST /api/auth/register` — 注册

```json
// 请求体
{
  "username": "zhangsan",       // 2–50 字符，唯一
  "email": "zhangsan@qq.com",
  "password": "123456"          // 至少 6 位
}

// 响应 (201)
{
  "token": "eyJhbGciOiJIUzI1NiIs...",
  "user": {
    "id": "uuid-string",
    "username": "zhangsan",
    "email": "zhangsan@qq.com",
    "created_at": "2026-07-12T10:30:00"
  }
}

// 错误: 400 — "用户名已被注册"
```

#### `POST /api/auth/login` — 登录

```json
// 请求体
{
  "username": "zhangsan",
  "password": "123456"
}

// 响应 (200) — 格式同注册

// 错误: 401 — "用户名或密码错误"
```

#### `GET /api/auth/me` — 获取当前用户信息

**需要认证。**

```json
// 响应 (200)
{
  "id": "uuid-string",
  "username": "zhangsan",
  "email": "zhangsan@qq.com",
  "created_at": "2026-07-12T10:30:00"
}
```

### 5.3 项目管理接口

**路由前缀：** `/api/projects`　|　**需要认证。**

#### `GET /api/projects` — 获取项目列表

```json
// 响应 (200)
[
  {
    "id": "proj-uuid",
    "user_id": "user-uuid",
    "name": "我的新家",
    "owner_name": "我",
    "current_stage_id": "design",
    "created_at": "2026-07-10T08:00:00"
  }
]
```

#### `POST /api/projects` — 创建项目

```json
// 请求体
{
  "name": "新房子装修",
```

## 6. 数据库结构

系统使用 MySQL 数据库 `renovation`，共 14 张表：

```
users
  ├── projects
  │     ├── budgets               (1:1)
  │     ├── budget_categories     (1:N)
  │     ├── todos                 (1:N)
  │     ├── expenses              (1:N)
  │     ├── flow_progress         (1:1)
  │     ├── price_categories      (1:N)
  │     │     └── price_models    (1:N)
  │     │           └── channel_quotes  (1:N)
  │     ├── selected_purchases    (1:N)
  │     └── synced_models         (1:N)
  │
  └── (全局共享表)
        ├── purchase_ref_stages
        │     └── purchase_ref_subgroups
        │           └── purchase_ref_items
```

所有表由 SQLAlchemy 在首次启动时通过 `Base.metadata.create_all` 自动创建。

### 关键表字段速查

| 表名 | 主要字段 |
|------|----------|
| `users` | id, username(唯一), email, password_hash, created_at |
| `projects` | id, user_id(FK), name, owner_name, current_stage_id, created_at |
| `budgets` | project_id(PK/FK), total |
| `budget_categories` | id, project_id(FK), name, color, allocated, spent |
| `todos` | id, project_id(FK), title, stage_id, due_date, completed, created_at |
| `expenses` | id, project_id(FK), title, amount, category_id, stage_id, date, status, payer, note |
| `flow_progress` | project_id(PK/FK), flow_type, done_step_ids(JSON), custom_order(JSON) |
| `price_categories` | id, project_id(FK), name, icon |
| `price_models` | id, category_id(FK), name, spec, note, quantity |
| `channel_quotes` | id, model_id(FK), channel, price, url, updated_at |
| `purchase_ref_items` | id, subgroup_id(FK), name, spec, qty, unit |

## 7. 常见问题

### Q: 是否必须注册账号？

不需要。所有本地功能（预算、待办、记账、流程、比价、采购）均可离线使用。注册账号后可以使用云端数据同步功能，在多台设备间同步数据。

### Q: 数据存储在什么地方？

浏览器 localStorage（键名 `xiaozhuangjia_state_v1`），同时支持导出为 JSON 文件。登录后可将数据上传到服务器 MySQL 数据库。

### Q: 如何迁移数据到另一台电脑？

三种方式：
1. 登录账号后上传到云端，在另一台电脑登录后从云端恢复
2. 使用「导出 JSON」下载文件，在新电脑上「导入 JSON」
3. 使用记账模块的 CSV 导出/导入功能

### Q: 如何修改默认的 5 个预算分类？

当前版本预算分类为固定 5 项，不支持新增或删除分类，但可以自由调整每个分类的分配金额。

### Q: 前端端口被占用怎么办？

编辑 [frontend/vite.config.ts](../frontend/vite.config.ts)（如不存在则创建）：

```typescript
export default defineConfig({
  server: { port: 5174 }
});
```

### Q: 后端数据库连接失败？

1. 确认 MySQL 服务正在运行
2. 确认数据库 `renovation` 已创建
3. 确认 `DATABASE_URL` 中的用户名、密码、主机地址、端口正确
4. 测试连接：`mysql -h 192.168.31.146 -P 3307 -u renovation -p renovation`

### Q: 如何重置所有数据？

在「工具」页面中，使用「数据管理」区域的「重置数据」按钮。此操作会清除当前项目所有数据并恢复默认设置，**操作不可逆**，建议先导出备份。

### Q: 忘记了密码怎么办？

当前版本暂未提供密码找回功能。请联系管理员在数据库中重置密码，或重新注册新账号。

### Q: 支持哪些浏览器？

推荐使用最新版 Chrome、Edge、Firefox、Safari。需支持 ES2020+ 和 CSS Grid/Flexbox。
