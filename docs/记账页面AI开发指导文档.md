# 记账页面 AI 开发指导文档

## 1. 目标

本文件用于指导后续 AI 或开发者基于当前 React + TypeScript 代码，继续完善记账页面的交互与视觉效果。

## 2. 参考目标

参考附件中的记账参考页结构，结合当前项目中的记账页面实现，目标是让页面在现有金额与分类总计模块下方，提供三种可点击视图：

- 分类统计
- 全部账单
- 设置

其中：

- 分类统计视图用于展示分类汇总信息。
- 全部账单视图保留当前已有的筛选、导入/导出、记一笔等操作逻辑。
- 设置视图用于承接后续分组管理、显示开关、分类配置等扩展能力。

## 3. 当前实现位置

- 页面主体：frontend/src/pages/ExpensePage.tsx
- 页面样式：frontend/src/styles/components.css

## 4. 页面结构要求

### 4.1 顶部摘要区

保留现有的以下内容：

- 总支出卡片
- 各分类总计卡片

### 4.2 视图切换栏

在上述卡片下方增加一个切换栏，视觉上参考参考页中的 toolbar-row 设计，采用以下三个按钮：

- 分类统计
- 全部账单
- 设置

要求：

- 按钮可点击，当前选中态有明显高亮。
- 交互使用 `button` 元素，便于支持键盘访问。
- 样式尽量与现有页面保持一致，使用柔和圆角、边框和浅色背景。

### 4.3 全部账单视图

全部账单视图需要保持现有的筛选和操作区不变，重点包括：

- 状态筛选按钮：全部/已支付/预付款/未支付
- 搜索输入框
- 导出/导入按钮组
- 记一笔按钮

也就是当前页面中的 expense-toolbar 结构应保持其原有逻辑与布局，不要破坏现有交互。

### 4.4 设置视图

设置视图可作为后续拓展入口，建议先展示以下内容：

- 分类显示说明
- 账单操作说明
- 可跳转到分类统计或全部账单的入口按钮

后续可以继续拓展为：

- 按类别分组显示
- 自定义分类显示开关
- 分组拖拽管理
- 主页面预算/统计展示开关

## 5. 交互建议

### 5.1 视图切换

建议用 `activeView` 状态管理当前视图：

- `stats`
- `bills`
- `group`

### 5.2 统计展示

分类统计页可以在后续扩展为：

- 按分类汇总金额
- 按状态汇总金额
- 按阶段汇总金额

### 5.3 账单列表

全部账单页应继续保留当前折叠列表与记录项详情展示。

## 6. 样式建议

建议参考如下视觉要素：

- 顶部切换栏使用轻边框与圆角
- 当前激活按钮使用主题色高亮
- 设置区块使用卡片式布局，包含标题、说明和按钮组
- 保持与现有页面的暖色系和圆角风格统一

## 7. 验收要求

- 在总支出和分类总计组件下方显示可点击的切换栏。
- 点击“全部账单”时，现有筛选和操作区保持可用。
- 点击“设置”时，显示一个可继续扩展的设置面板。
- 页面构建无报错。

## 8. 实现备注

当前版本已经完成基础切换栏与设置面板结构，后续可继续把“分类统计”内容扩展为更接近参考页的统计视图，并把“设置”页发展为真正的分组管理页面。
